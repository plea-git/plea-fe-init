### 기본 Layout

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 영역 구분 | 기본 Layout은 상단 영역과 하단 영역으로 구분되어 노출된다. | |
| GNB 영역 | 상단 영역은 GNB 영역으로, 각 고객사 CI, 메뉴, 접속자 정보가 노출된다. | |
| CI 클릭 | GNB 영역 내 고객사 CI 클릭 시 메인페이지로 이동된다. | |
| 메뉴 클릭 | GNB 영역 내 각 메뉴 클릭 시 그에 해당하는 메뉴화면으로 이동된다. | |
| Body 영역 | 하단 영역은 Body 영역으로, Contents 영역과 Footer 영역으로 구분된다. | |

| Footer 영역 | Footer 영역은 메인, 로그인 화면 하단에 고정되어 노출된다. | 별도 정의 없는 경우 메인, 로그인 화면만 노출 |

## 2. 화면이동

### 뒤로가기

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 작성중 뒤로가기 | (기획안에 별도 표기된 작성중 화면) 에서 저장하지 않고 뒤로가기 시 컨펌 얼럿이 출력된다. | ex) 알파키 홈페이지 청약 신청, KB 메시지의뢰 작성 |
| 세션 만료 후 뒤로가기 | 세션 만료 후 뒤로가기 시 로그인 화면으로 이동된다. | |
| 브라우저 뒤로가기 | 브라우저 뒤로가기 버튼 클릭 시 이전페이지로 이동된다. | |
| 상태 복원 | 동일 메뉴 내 뒤로가기 시 이전 정렬/검색/선택 상태를 복원하여 이전 페이지가 노출된다. | 목록 / 상세 페이지 간 뒤로가기 등 |

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|

| 스크롤 초기화 | 페이지 새로고침 시 스크롤 포커싱은 상단으로 이동된다. | |
| 검색조건 유지 | 페이지 새로고침 시 검색조건, 정렬, 노출개수(Page Count)는 유지되나 체크박스 선택값은 초기화된다. | 목록페이지에 대한 정의 사항. 등록페이지는 기획안에 따름 |

### 세션

|----------|----------|------|

## 3. 컴포넌트

### 체크박스

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 기본 상태 | 기본 상태의 체크박스는 디폴트 값으로 Unchecked 상태로 노출된다. | 기본상태: Not (disable + read only) |
| Unchecked → Checked | Unchecked 된 기본 상태의 체크박스는 클릭 시 Checked 상태로 변경된다. | |
| Checked → Unchecked | Checked 된 기본 상태의 체크박스는 클릭 시 Unchecked 상태로 변경된다. | |
| 비활성화 | 비활성화 (disable)된 체크박스는 클릭 시 선택 상태가 변경되지 않는다. | |
| 읽기전용 | 읽기전용 (Read Only)된 체크박스는 클릭 시 선택 상태가 변경되지 않는다. | |
| 일부 Unchecked | 테이블 내 하위 체크박스 일부만 Unchecked 된 경우 전체 체크박스 또한 Unchecked 상태로 노출된다. | |
| 일부 Checked | 테이블 내 하위 체크박스 일부만 Checked 된 경우 전체 체크박스는 [-] 상태로 노출된다. | |
| 전체 Checked | 테이블 내 하위 체크박스가 모두 Checked 된 경우 전체 체크박스 또한 Checked 상태로 노출된다. | |
| 전체 Unchecked 변경 | 전체 체크박스를 Unchecked 상태로 변경할 시 기본 상태의 하위 체크박스는 모두 Unchecked 상태로 변경된다. | |
| 전체 Checked 변경 | 전체 체크박스를 Checked 상태로 변경할 시 기본 상태의 하위 체크박스는 모두 Checked 상태로 변경된다. | |

### 라디오 버튼

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 기본 상태 | 라디오 버튼은 기본적으로 Off 상태로 노출된다. | |
| On 변경 | 기본 상태의 라디오 버튼을 클릭했을 시 해당 라디오 버튼은 On 상태로 변경된다. | |
| 상호 배타 | 기본 상태의 라디오 버튼이 On 상태로 변경될 시, 연관된 나머지 라디오 버튼은 Off 상태로 변경된다. | |
| 비활성화 | 비활성화 (disable)된 라디오버튼은 색상 처리가 되어있고, 클릭 시 선택 상태가 변경되지 않는다. | |
| 읽기전용 | 읽기전용 (Read Only)된 라디오버튼은 색상 처리가 되어있고, 클릭 시 선택 상태가 변경되지 않는다. | |

| 첫 페이지 | 제일 첫 페이지 이동 버튼 클릭 시 첫 페이지를 기준으로 페이지번호가 탐색된다. | |
| 마지막 페이지 | 제일 마지막 페이지 이동 버튼 클릭 시 마지막 페이지를 기준으로 페이지번호가 탐색된다. | |

### 칩

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 칩 팝업 | 칩 타이틀 선택 시 칩 선택 팝업이 노출된다. | |
| 선택 개수 | 칩 선택 팝업을 통해 선택된 개수에 따라 칩 타이틀 옆에는 선택한 개수가 추가 노출된다. | |
| 미선택 | 선택한 항목이 없는 경우, 칩 타이틀 옆 선택한 개수는 미노출 된다. | |
| 선택 항목 노출 | {서비스에 따라 선택 항목 노출로 기획된 경우} 선택한 칩이 타이틀 우측에 노출되며 지정된 영역을 초과할 시 말줄임 처리된다. | |
| 선택 항목 미노출 | {서비스에 따라 선택 항목 미노출로 기획된 경우} 선택한 칩은 노출되지 않으며, 타이틀 및 선택 개수만 표기된다. | |
| 초기화 | [초기화] 버튼 클릭 시 전체 선택 해제된다. | |

### 칩 선택 팝업

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 팝업 구성 | 칩 선택 팝업은 타이틀, "닫기" 버튼, 항목 리스트 영역과 고정영역이 출력된다. | |
| 기존 선택 유지 | 기존에 선택된 칩이 있는 경우, 칩 선택 팝업이 노출될 시 해당 항목은 선택 상태로 노출된다. | |
| 스크롤 | 항목 리스트 영역을 초과하여 항목이 노출된 경우, 스크롤이 생성된다. | |
| 항목 선택 | 미선택 항목을 클릭할 시 선택한 항목으로 변경되며 "X" 버튼이 노출된다. | |
| 항목 해제 | 선택한 항목을 클릭 또는 "X" 버튼을 클릭할 시 미선택 항목으로 변경된다. | |
| 고정영역 구성 | 고정영역은 [초기화] 및 [적용] 버튼으로 구성된다. | |
| 초기화 | [초기화] 버튼 클릭 시 전체 항목 선택 해제되어 미선택 항목으로 변경된다. | |
| 적용 개수 | [적용] 버튼은 선택된 항목에 따라 옆에 선택한 개수가 노출된다. | |
| 적용 비활성화 | [적용] 버튼은 선택된 항목이 없을 경우, 비활성화 된다. | |
| 적용 활성화 | [적용] 버튼은 1개 이상의 항목이 선택된 경우 활성화 된다. | |
| 재비활성화 | 1개 이상 선택 후 모두 선택 해제 했을 시 [적용] 버튼은 재차 비활성 상태로 변경된다. | |

## 4. 입력

### 인풋

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 필수값 표기 | 필수값인 인풋은 '*' 표기를 통해 강조처리 된다. | |
| 플레이스홀더 | 미입력된 인풋은 플레이스홀더 문구가 노출된다. | |
| 입력값 노출 | 입력값이 존재하는 인풋은 플레이스홀더 문구 대신 입력값이 노출된다. | 입력값 존재한 경우 |
| Focus 상태 | Focused 된 인풋은 테두리 색상 강조되며 focus된 위치가 표시된다. | |
| 비활성화 | 비활성화된 인풋은 수정 불가 상태로 노출되며 입력값 유무에 따라 플레이스홀더 문구 또는 입력값이 노출된다. | |
| 읽기전용 Type 1 | 읽기전용인 인풋 Type 1 - 인풋박스 노출되며 이전 입력값 또는 플레이스 홀더 문구가 노출된다. | 입력값 글씨 색상값 기획 재확인 후 해당 내용 전달 예정 |
| 읽기전용 Type 2 | 읽기전용인 인풋 Type 2 - 별도 인풋박스 없이 해당 항목에 대한 값이 노출된다. | |

### 인풋 (Search)

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 플레이스홀더 | 미입력된 인풋은 플레이스홀더 문구가 노출된다. | |
| 검색 실행 | 돋보기 Icon 클릭 또는 Focus 후 엔터키 입력 시 인풋에 대한 검색 요청된다. | |
| X 아이콘 | 입력값이 존재하는 인풋에 관해 Focus out 상태에선 입력값 우측에 X 아이콘 노출된다. | |
| 입력값 삭제 | 입력값이 존재하는 인풋의 X 아이콘 클릭 시 입력값이 사라지며 플레이스 홀더 문구가 노출된다. | |
| 비활성화 | 비활성화된 인풋은 돋보기 클릭해도 검색 불가하다. | |
| 읽기전용 | 읽기전용인 인풋은 엔터 또는 돋보기 클릭해도 검색 불가하다. | |

### 인풋 유효성 체크

| 시험항목 | 시험내역 | 비고 |
|----------|----------|------|
| 에러 메시지 노출 | 에러 메시지는 입력폼 입력 후 Focus Out 시 노출된다. | |
| 데이터 검증 | 데이터 검증은 입력폼 입력 후 Focus Out 시 이루어진다. | validation |
| 즉시 검증 | 데이터 검증 예외항목으로 즉시성으로 인풋란에 변경이 있는 경우에는 입력 시 유효성 오류가 표기된다. | ex) 숫자만 입력 가능하지만 한글 입력 |
| 에러 색상 | 에러 메시지는 색상 강조가 적용된다. | |

### 달력
